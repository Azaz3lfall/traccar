1 INTRODUÇÃO
Este documento descreve todo o protocolo de comunicação ASCII entre os rastreadores RST,
para versões de firmware superior ou igual a V4.14, e o servidor de comunicação. Para a
comunicação entre o rastreador e o servidor é usado o protocolo UDP. Atualmente, o rastreador não
suporta a comunicação TCP.
É possível utilizar uma versão reduzida do protocolo ASCII, no final do documento tem o
detalhamento desse formato.

3 COMUNICAÇÃO ENTRE O RASTREADOR E O SERVIDOR

3.1 COMUNICAÇÃO DO RASTREADOR PARA O SERVIDOR
Quando o rastreador envia uma mensagem para o servidor, o rastreador irá aguardar o
recebimento do ACK da mensagem do servidor para só depois incrementar o seu número de
sequência e enviar uma nova mensagem. O servidor deverá sempre responderá o ACK (ID: 6) ou
NACK (ID: 21) com o número de sequência recebido e o número de série do equipamento. Caso o
rastreador não receba o ACK ou NACK da mensagem enviada, o mesmo irá tentar enviar a
mensagem novamente, só que aumentando o tempo de timeout de resposta do servidor. O limite
máximo para o envio da mensagem para cada número de IP é de 5 tentativas para cada número IP.
Após todas as tentativas, o rastreador automaticamente reinicia toda a comunicação GPRS.
É imprescindível que o servidor responda a mensagem recebida com ACK ou NACK para o
rastreador, pois se não houver uma resposta do servidor, o mesmo irá tentar enviar a mesma
mensagem sempre. Porém a partir da versão de firmware V2.02, o rastreador vai desistir de enviar
uma mensagem para o servidor após 20 tentativas (ou o valor configurado) de envio e nenhum
recebimento de ACK ou NACK. Este valor de tentativas poderá ser configurado conforme
necessidade do cliente.

3.2 COMUNICAÇÃO DO SERVIDOR PARA O RASTREADOR
Quando o servidor envia uma mensagem para o rastreador, o rastreador irá responder com
uma mensagem de ACK (ID: 134) ou NACK (ID: 149) para o servidor. O rastreador irá responder
todas as mensagens recebidas do servidor com o mesmo número de sequência que o servidor
enviou. Sendo assim, o servidor terá o seu próprio número de sequência e o rastreador terá outro.

5.2 MENSAGEM ID: 2 – TRACKING EM SLEEP
Esta mensagem informa ao servidor a posição atual do veículo, quando o rastreador estiver em

arquivo do novo firmware.

8.1 FORMA DE FUNCIONAMENTO DA ATUALIZAÇÃO
Ao iniciar a atualização remota, o rastreador ficará responsável por solicitar os pacotes,
verificação de erro dos dados, validação do CRC do firmware novo, backup do firmware atual e reset
automático para atualização do firmware, sendo que o servidor só será responsável por enviar o
comando de início de atualização, responder as solicitações com os dados do firmware novo e o
comando de cancelamento da atualização.

Mensagem ID 18: Solicitação do pacote de dados do firmware



8.4 PASSO A PASSO PARA ATUALIZAÇÃO
Abaixo é demonstrado os passos que devem seguir para a correta atualização de firmware do
rastreador:
1) Com a ignição ligada, o servidor deverá enviar a mensagem de início de atualização de
firmware (ID: 138);
2) Após o rastreador receber esse comando, ele irá solicitar os pacotes da nova versão de
firmware através da mensagem ID: 18;
3) O servidor ao receber a mensagem ID 18, deverá ler qual é o número do pacote solicitado e
responder ao rastreador com os dados utilizando a mensagem ID: 139;
4) Ao termino do envio de todos os pacotes corretamente, o rastreador irá enviar a mensagem
ID 19 com o valor do status em 1, indicando o final das solicitações dos pacotes;
5) Nesse momento o rastreador irá automaticamente se reiniciar em 30 segundos para trocar
a versão de firmware;
6) Ao voltar a comunicar-se com o servidor, o rastreador irá enviar a mensagem ID 19 com o
valor do status em 5;
7) Nesse momento a atualização de firmware foi concluída com sucesso.

Em qualquer momento da atualização é possível enviar o comando para cancelar a atualização
(ID: 140).
